Built Using...

• Swift
• WatchOS UI Kit
• Timers


This project taught me a lot about how to develop UI for very limited space. I had some idea of a limited interface from general mobile development, but WatchOS UI Kit doesn't even offer contraints for UI objects. This made me really focus on the design and make tough decisions on what was necessary vs. what I thought would look cute.


I also learned the struggle of starting and stopping timers. To keep a tempo with the music, I had to manage changes with timers while still allowing that tempo to be change to the artist's preference. Timers can be a headache!



What's Coming Next...

To make this project better, I would like to:


  • Create a scrolling effect rather than just having the chords change with no warning. This would help musicians keep time and prepare for incoming changes before they happen.
  • Create a companion phone app that offers an interface for users to input songs and chords. This would remove the barrier of currently hardcoding chords for songs.
  • Network the companion app so that users can start to create a shared database of songs that anyone can use on their watch.
